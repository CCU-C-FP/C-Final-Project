# 排球戰術數據記錄系統 - 架構設計

本文檔描述目前核心邏輯層的實作架構、資料流與各元件職責。此專案已完成可獨立運作的核心邏輯，UI 層仍可在此基礎上擴充。

## 設計目標

- 將資料模型、業務邏輯與輸出工具分層，降低耦合。
- 以事件驅動方式讓 UI 或其他消費者同步比分與事件狀態。
- 以強型別 enum 取代字串判斷，減少錯誤。
- 支援撤銷 / 重做，讓事件紀錄與比分保持一致。
- 保持無外部依賴，便於跨平台執行核心邏輯。

## 核心資料流

### 1. 建立比賽上下文

`Team`、`Player` 和 `Match` 先建立比賽的基本狀態，包含雙方隊伍、球員、場地、局數與比賽狀態。

### 2. 記錄事件

`EventManager.AddEvent` 負責把 `GameEvent` 放進事件序列，同時推入 undo 棧，並觸發 `EventAdded`。

### 3. 自動計分

`ScoringService.ProcessGameEvent` 根據事件類型判斷是否得分，再更新 `Match` 內的隊伍局分。

### 4. 同步撤銷 / 重做

當 `EventManager` 的事件歷史改變時，`ScoringService` 會重新計算所有比分，避免事件清單與比分狀態脫節。

### 5. 統計與導出

`StatisticsEngine` 直接以事件清單計算成功率、得分來源、失誤集群與趨勢資料；`CsvExporter` 再把這些結果輸出成 CSV。

## 元件職責

### Models

這一層只保存資料與基本行為：

- `ActionType` 描述所有事件類型。
- `MatchStatus` 描述比賽生命週期。
- `TeamSide` 區分主隊與客隊。
- `GameEvent` 保存單筆事件快照。
- `Player` 保存球員資料與所屬隊伍。
- `Team` 保存球隊、局分與球員名單。
- `Match` 保存整場比賽狀態與當前比分。

### Services

#### EventManager

- 維護事件清單。
- 提供 `Undo`、`Redo` 與查詢功能。
- 透過 `EventAdded`、`EventUndone`、`EventRedone`、`EventsCleared` 通知外部。

#### ScoringService

- 接收事件後更新比分。
- 處理單局 25 分先勝且需差 2 分的規則。
- 處理三先勝制比賽結束條件。
- 提供比分文字與事件通知給 UI。

#### StatisticsEngine

- 提供球員統計，且以 `playerId + TeamSide` 為統計鍵。
- 提供隊伍成功率、得分來源與失誤來源分析。
- 提供比分進度曲線與失誤密集點分析。
- 提供文字型報告輸出。

### Utilities

#### CsvExporter

- 輸出事件清單 CSV。
- 輸出統計報告 CSV。
- 內建 CSV 轉義，處理逗號、引號與換行。

## 重要設計決策

### 事件驅動解耦

服務層不直接依賴 UI。UI 只需要訂閱事件，例如比分更新或事件新增，就能刷新畫面。

### 強型別事件模型

事件類型、隊伍標識與比賽狀態都用 enum 表達，避免用字串做條件判斷。

### 個人統計需要隊伍上下文

因為雙方球員可能有相同背號，所以所有球員統計 API 都同時要求 `playerId` 與 `TeamSide`。

### 可重算的比分狀態

比分不是只靠單一步驟累加，而是可以根據事件歷史完整重算，這是撤銷 / 重做能保持正確的關鍵。

## 執行時流程

1. 建立 `Match`，並設定 `HomeTeam`、`AwayTeam`。
2. 建立 `EventManager`、`ScoringService`、`StatisticsEngine`。
3. 訂閱 `ScoreUpdated`、`EventAdded` 等事件。
4. 每次發生動作時建立 `GameEvent`，先加入事件管理器，再交由計分服務處理。
5. 若使用者撤銷或重做，事件管理器觸發變更，計分服務重新計算比分。
6. 比賽完成後，統計引擎與 CSV 匯出可直接使用同一份事件資料。

## 擴充點

- 新增 UI 時，只要訂閱現有事件，不需要改核心邏輯。
- 新增統計指標時，可在 `StatisticsEngine` 上延伸，不影響事件管理與計分。
- 若之後加入資料庫或雲端同步，可優先放在 `Utilities` 或新增獨立儲存層。

## 目前範圍

- 核心邏輯已可執行與測試。
- WinForms UI 尚未在此版本中實作。
- 專案目前仍以單機內存資料流為主，未引入資料庫或外部服務。
